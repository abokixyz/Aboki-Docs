# Add Token Support

Configure which tokens your customers can receive on your platform. You can add custom tokens beyond the default ones (USDC, USDT, ETH, SOL).

<ParamField header="Authorization" type="string" required>
  Bearer token from your business login. Format: Bearer {your_jwt_token}
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be application/json
</ParamField>

## Body Parameters

<ParamField body="network" type="string" required>
  Blockchain network. Choose: base, solana, or ethereum
</ParamField>

<ParamField body="tokens" type="array" required>
  Array of token objects to add
</ParamField>

### Token Object Fields

<ParamField body="tokens[].address" type="string" required>
  Token contract address (e.g., 0x833589fcd6edb6e08f4c7c32d4f71b54bda02913)
</ParamField>

<ParamField body="tokens[].symbol" type="string" required>
  Token symbol (e.g., USDC, CUSTOM, MYTOKEN)
</ParamField>

<ParamField body="tokens[].name" type="string" required>
  Full token name (e.g., USD Coin, My Custom Token)
</ParamField>

<ParamField body="tokens[].decimals" type="number" required>
  Token decimals. Usually 6 for USDC/USDT, 18 for most others
</ParamField>

<ParamField body="tokens[].feePercentage" type="number" required>
  Your fee percentage. Example: 1.5 for 1.5%
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<ResponseField name="data" type="object">
  Contains details about the added tokens
</ResponseField>
