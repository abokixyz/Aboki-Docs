---
title: "Add Custom Tokens"
api: "POST https://aboki-b2b-eobk.onrender.com/api/v1/business/tokens/add"
description: "Add new custom tokens that users can trade to on your platform (in addition to default tokens)"
---

Add custom destination tokens that your users can receive when using your platform. This is in addition to the default tokens (USDC, USDT, ETH, SOL) that are automatically provided.

<ParamField header="Authorization" type="string" required>
  Bearer JWT token from business login. Format: `Bearer <your_jwt_token>`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Body Parameters

<ParamField body="network" type="string" required>
  Blockchain network. Must be one of: `base`, `solana`, `ethereum`
</ParamField>

<ParamField body="tokens" type="array" required>
  Array of token objects to add
  <Expandable title="Token Object Structure">
    <ParamField body="address" type="string" required>
      Token contract address (e.g., `0x833589fcd6edb6e08f4c7c32d4f71b54bda02913`)
    </ParamField>
    <ParamField body="symbol" type="string" required>
      Token symbol (e.g., `CUSTOM`, `MYTOKEN`)
    </ParamField>
    <ParamField body="name" type="string" required>
      Full token name (e.g., `Custom Token`, `My Custom Token`)
    </ParamField>
    <ParamField body="decimals" type="number" required>
      Token decimals (typically 6, 8, or 18)
    </ParamField>
    <ParamField body="feePercentage" type="number" required>
      Fee percentage you want to charge (e.g., 1.5 for 1.5%)
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable success message
</ResponseField>

<ResponseField name="data" type="object">
  Contains the added token information and business details
</ResponseField>

Important Notes
<Note>
You must have a valid JWT token from creating and logging into your business account first.
</Note>
<Warning>
Make sure to verify the token contract address is correct before adding. Invalid addresses will be rejected.
</Warning>
<Tip>
You can add multiple tokens in a single request by including them in the tokens array.
</Tip>
